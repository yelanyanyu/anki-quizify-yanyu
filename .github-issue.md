## Description

When a card template's `<script>` block contains JavaScript strings with HTML
closing tags — such as `'</div>'`, `'</span>'`, `'</audio>'` — the **entire
`<script>` block is silently discarded** by Anki's renderer. No syntax error is
thrown; the script simply never executes. This causes downstream `undefined`
errors for any symbols defined in that block.

The same code works correctly in a standalone browser (Chrome/Firefox), ruling
out a JavaScript-level issue.

## Steps to Reproduce

1. Create a note type with a Front field and add this minimal template as the
   front side:

```html
<div id="status" style="color:#fff;background:#111;padding:4px;"></div>

<!-- Script A: works fine -->
<script>
document.getElementById('status').textContent += ' [A:OK]';
</script>

<!-- Script B: silently discarded because the string contains </div> -->
<script>
document.getElementById('status').textContent += ' [B:OK]';
function harmless() {
    return '<div>test</div>';  /* </div> inside a string breaks this block */
}
document.getElementById('status').textContent += ' [B:END]';
</script>

<!-- Script C: works fine again -->
<script>
document.getElementById('status').textContent += ' [C:OK]';
</script>
```

2. Preview the card in Anki.
3. Observe the status text.

## Expected Behavior

The status text should show: `[A:OK] [B:OK] [B:END] [C:OK]`

## Actual Behavior

The status text shows: `[A:OK] [C:OK]`

Script B is entirely missing — neither `[B:OK]` nor `[B:END]` appears.

## Bisect Evidence

We performed extensive incremental testing on this template with different
combinations of functions inside and outside IIFEs:

| Functions in one `<script>` block | Result |
|---|---|
| `parseExamples` only (8 lines) | ✅ runs |
| + `buildItemHTML` (20 lines) | ✅ runs |
| + `buildControls` (35+ lines, Chinese strings) | ❌ discarded |
| Same `buildControls` with English strings | ❌ discarded |
| All 3 helpers in **separate** `<script>` blocks | ✅ all run |
| `buildItemHTML` with DOM APIs (no `</...>` strings) | ✅ runs |
| `buildControls` with DOM APIs (no `</...>` strings) | ✅ runs |

The problem is triggered by HTML closing tags inside JavaScript string literals.
Additionally, there is a **cumulative effect**: more functions / longer script
blocks make the discard more likely, even with identical string content.

## Workaround (Confirmed)

Replace string-based HTML generation with DOM APIs:

```javascript
// ❌ Discarded by Anki
var html = '<div class="item"><span class="text">' + text + '</span></div>';

// ✅ Works
var div = document.createElement('div');
div.className = 'item';
var span = document.createElement('span');
span.className = 'text';
span.textContent = text;
div.appendChild(span);
var html = div.outerHTML;
```

Alternatively, splitting functions across multiple small `<script>` blocks
mitigates the issue.

## Additional Context

- The Anki reviewer pipeline sets card HTML via `.innerHTML`, then manually
  extracts and re-injects `<script>` tags (see `reviewer/index.ts` and
  [issue #1120](https://github.com/ankitects/anki/issues/1120)). This
  extraction/re-injection step is the likely point where the HTML parser's tag
  matching interferes with script content.
- The Quizify template author already worked around this by splitting code into
  12+ tiny `<script>` blocks, suggesting this bug has been present for a long
  time.
- Similar bugs exist in other tools (WordPress #54179, PHP #63417, jQuery
  #13821) where HTML closing tags inside script strings confuse DOM parsers.
- The issue is **not reproducible** in a standalone browser — it is specific to
  Anki's rendering pipeline.
