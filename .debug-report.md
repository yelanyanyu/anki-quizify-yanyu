# Anki NestedReveal 调试报告

## 现象

在 Anki 25.09.2 (Qt 6.9.1 / Chromium 122) 中，`QuizifyNestedReveal` 模块始终为 `undefined`，所有新功能完全失效。同样的代码在 test.html（浏览器）中一切正常。

## 通过二分法定位到的边界

| 版本 | 内容 | 结果 |
|------|------|------|
| v3 | 3 行极简 IIFE | ✅ 通过 |
| v4 | `parseExamples` 在 IIFE 内 | ✅ 通过 |
| v5 | + `buildItemHTML` 在 IIFE 内 | ❌ 崩溃 |
| v6 | `buildItemHTML` 独立函数 + 极简 IIFE | ✅ 通过 |
| v8 | 仅 `__nestedParseExamples` helper | ✅ 通过 |
| v9 | + `__nestedBuildItemHTML` helper | ✅ 通过 |
| v10 | + `__nestedBuildControls`（中文按钮） | ❌ 崩溃 |
| v11 | + `__nestedBuildControls`（英文按钮） | ❌ 崩溃 |

## 关键发现

1. **不是中文字符的问题**：v11 用英文替换中文后仍然崩溃。
2. **不是特定 HTML 标签的问题**：`buildItemHTML` 包含 `</span>`、`</audio>`、`</div>` 也能通过。
3. **不是 IIFE 的问题**：将函数移到 IIFE 外面（独立 helper）后，到第三个 helper 仍然崩溃。
4. **与 script 块位置无关**：无论放在模板最前面还是 `{{/Skip Replace}}` 之后，结果相同。
5. **疑似累计代码量阈值**：每多加一个 helper 函数，script 块总行数增加约 15-20 行。到第三个 helper 时总代码量约 40 行时崩溃。原始模板将几乎所有逻辑拆分到极小的 `<script>` 块中，可能有意为之。

## 预测

Anki 的 Qt WebEngine 对单个 `<script>` 块可能有行数或 AST 节点数限制（不是字符数，原始模板有更大块），超过后整个块被丢弃。原始模板作者把代码拆成 12 个独立 `<script>` 块可能就是踩过这个坑。

## 下一步

将 `__nestedBuildControls` 拆成独立的小函数/独立 script 块，验证这个假说。
